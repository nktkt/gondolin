    Random #

    Deterministic RNG utilities for Gondlin (seed-threaded, pure).

    Why not IO.rand / runtime randomness? #

    Lean (and mathlib) can generate random numbers via IO, but that gives effectful randomness whose results depend on hidden runtime state. For Gondlin, that is a poor fit:

    • it breaks the “one semantics” contract for compilation/verification (graphs stop being a pure mathematical object unless you model the RNG state explicitly),
    • it makes replays / certificate checking brittle (you want reproducible runs),
    • it complicates the proof-linked compilation path (effects and mixed dtypes).

    Instead we use a deterministic pseudorandom generator and treat “randomness” as a deterministic function of an explicit seed (and a counter/stream id). This mirrors the JAX/functional RNG style and keeps the semantic core pure.

    What this file provides #

    • a small 64-bit PRNG (SplitMix64-style mixing),
    • deterministic sampling utilities keyed by (seed, counter, linearIndex),
    • a deterministic way to build dropout-style {0,1} masks as shape-indexed tensors.

    Today this is used by Session-level stochastic layers (e.g. Gondlin.Session.dropout), where the RNG state is stored in NatRefs and used to generate masks reproducibly.

    If you want “PyTorch-like” randomness at the boundary, prefer sampling an initial seed in IO and then using the seeded RNG from that point onward (Gondlin.Session.initRngFromIO).

                        Standard normal tensors #

                        source
                        def Runtime.Autograd.Gondlin.Random.boxMullerCos {α : Type} [Context α] (u1 u2 : α) :
                        α

                        Box–Muller transform: turn two independent uniforms u1,u2 ∈ (0,1) into a standard normal sample.

                        We return only the cos branch:

                        z = sqrt(-2 * log u1) * cos(2π * u2).

                        Notes:

                        • We clamp u1 below by ε to avoid log 0.
                        • This is intended as a deterministic pseudo-normal sampler for demos and benchmarking. It is not a cryptographic RNG.

                        Reference:

                        • Box & Muller (1958), "A Note on the Generation of Random Normal Deviates".
