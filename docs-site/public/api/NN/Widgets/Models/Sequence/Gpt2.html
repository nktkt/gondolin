    GPT-2 Training Log Viewer #

    Gondlin's generic TrainLog widget (#train_log_view / #train_log_file_view) is great for curves and small scalar summaries, but language-model examples benefit from a "prompt → sample" panel that keeps the generated text readable.

    This module provides GPT-2 specific viewers that:

    • extract prompt=... and generated=... notes (as written by the gondlin gpt2 example),
    • render them as separate monospace blocks, and
    • delegate the remaining metrics/notes to the generic training-log renderer.

    Usage (after running the executable to write a JSON log):

    #gpt2_train_log_file_view "data/model_zoo/gpt2_trainlog.json"

      One-shot Prompt Runner #

      #gpt2_prompt_view "..." runs the gondlin gpt2 executable (CUDA) with a small default configuration, writes a temporary JSON TrainLog, then renders it.

      This is meant for quick "does it run?" checks inside the infoview. It is not a replacement for the CLI interactive loop, and it will still take noticeable time if you increase --steps.
